# MAS Startup Eval

[Русский](README.md) | [English](README.en.md)

Мультиагентная система на Claude API для оценки перспективности российских технологических стартапов (0–3: закрылся / выживает без роста / растёт / лидер-exit).

## Данные

`Data/Startups.xlsx` — 352 стартапа (168 из топ-программы ПУЭТ + 184 контрольная группа), одна таблица, один лист. Поля: описание проекта, ключевые слова, рыночный/регуляторный контекст, потребность в инвестициях, метаданные команды, патентная и публикационная активность по теме (РФ + зарубеж, 2018–2024).

Реальный исход стартапа и принадлежность к группе (топ/контроль) в этом файле физически отсутствуют — хранятся отдельно, вне репозитория, и используются только для валидации после прогона, а не как второй лист в той же таблице.

## Структура

```
prompts/     системные промпты 5 агентов (Scout → Technologist/Market/Team → Integrator)
src/         пайплайн (src/pipeline.py)
Data/        Startups.xlsx — входная таблица со стартапами
results/     результаты прогона (JSON на стартап + results.csv, не в репо)
```

## Запуск

```
pip install -r requirements.txt
cp .env.example .env   # вписать ANTHROPIC_API_KEY

python src/pipeline.py --input Data/Startups.xlsx --limit 20
```

Флаги: `--input` (путь к таблице), `--output-dir` (по умолчанию `results`), `--limit` (ограничить число стартапов для отладки).

## Архитектура

Агенты не выходят в интернет и не видят реальный исход или принадлежность к группе — только поля датасета, явно перечисленные в белых списках в `pipeline.py`. Каждый агент возвращает JSON по фиксированной схеме (`output_config.format`), поэтому парсинг ответа надёжен без ретраев на кривой JSON.

| Агент | Получает из таблицы | + |
|---|---|---|
| Scout | Name, суть_проекта, ключевые_слова_nlp, рынок, основатель, вуз, регион, год | — |
| Technologist | pat_rus_\*, pub_\*, pat_foreign_\* | профиль Scout |
| Market | рынок, инвестиции, регион, год | профиль Scout |
| Team | основатель, вуз, регион, год | профиль Scout |
| Integrator | — | профиль Scout + выводы трёх специалистов |

## Валидация

`Уровень_исхода_0_3` (реальный исход, 0–3) и метка группы (топ/контроль) хранятся в отдельном приватном файле, не в `Data/`. После прогона сверяются с оценкой Интегратора: F1, recall, quadratic weighted kappa (порядковая шкала) — и сравнение оценок между топ- и контрольной группой (ключевая гипотеза исследования).
